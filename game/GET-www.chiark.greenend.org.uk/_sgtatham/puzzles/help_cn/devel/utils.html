第 5 章：实用工具 API

本章记录了 Puzzles 代码中供其他部分使用的各种实用工具 API。

5.1 随机数生成

不同平台的本地随机数生成器在质量和种子大小方面差异很大。因此，Puzzles 提供了自己的高质量随机数生成器，其额外优势是在不同平台上使用相同的种子数据时能产生相同的结果。这使得游戏随机种子可以在 Puzzles 的不同端口之间交换，并且仍然生成相同的游戏。

与 ANSI C 的 rand() 函数不同，Puzzles 随机数生成器有一个称为 random_state 的显式状态对象。例如，每个中端管理一个这样的对象，并将其传递给后端以生成游戏。

5.1.1 random_new()

random_state *random_new(char *seed, int len);

分配、初始化并返回一个新的 random_state。输入数据用作随机数流的种子（即，以后使用相同的种子将生成相同的流）。

种子数据可以是任何数据；不需要使用可打印的 ASCII、以 NUL 结尾的字符串或类似的东西。

5.1.2 random_copy()

random_state *random_copy(random_state *tocopy);

分配一个新的 random_state，将另一个 random_state 的内容复制到其中，并返回新状态。如果随后对副本和原始状态调用完全相同的函数序列，结果将是相同的。这对于使用给定的随机状态推测性地执行某些操作，以及稍后精确地重放该操作可能很有用。

5.1.3 random_free()

void random_free(random_state *state);

释放 random_state。

5.1.4 random_bits()

unsigned long random_bits(random_state *state, int bits);

返回一个从 0 到 2^bits-1 之间的随机数。bits 应该在 1 到 32 之间。

5.1.5 random_upto()

unsigned long random_upto(random_state *state, unsigned long limit);

返回一个从 0 到 limit-1 之间的随机数。limit 不能为零。

5.1.6 random_state_encode()

char *random_state_encode(random_state *state);

将 random_state 的整个内容编码为可打印的 ASCII。返回包含该编码的动态分配的字符串。随后可以将其传递给 random_state_decode() 以重建相同的 random_state。

5.1.7 random_state_decode()

random_state *random_state_decode(char *input);

解码由 random_state_encode() 生成的字符串，并重建与编码的 random_state 等效的 random_state，即它应该产生相同的随机数流。

此函数没有错误报告；如果传递无效字符串，它将简单地生成任意随机状态，这可能会变得明显非随机。

5.1.8 shuffle()

void shuffle(void *array, int nelts, int eltsize, random_state *rs);

将数组随机打乱顺序。接口与 ANSI C 的 qsort() 非常相似，只是不需要比较函数。

array 是指向数组第一个元素的指针。nelts 是数组中的元素数；eltsize 是单个元素的大小（通常使用 sizeof 测量）。rs 是用于为打乱过程生成所有随机数的 random_state。

5.2 预设菜单管理

函数 midend_get_presets()（第 4.17 节）返回一个描述菜单层次结构的数据结构。后端也可以选择向中端提供这样的结构，如果它们希望将预设按层次结构分组的话。为了使这变得容易，有一些实用函数可以构建预设菜单结构，还有一个供前端使用的函数。

5.2.1 preset_menu_new()

struct preset_menu *preset_menu_new(void);

分配一个新的 struct preset_menu，并将其初始化为不包含菜单项。

5.2.2 preset_menu_add_submenu()

struct preset_menu *preset_menu_add_submenu
    (struct preset_menu *parent, char *title);

将新子菜单添加到现有预设菜单的末尾，并返回指向描述该子菜单的新分配的 struct preset_menu 的指针。

字符串参数 ‘title’ 必须由调用者动态分配。预设菜单结构将取得其所有权，因此调用者不得释放它。

5.2.3 preset_menu_add_preset()

void preset_menu_add_preset
    (struct preset_menu *menu, char *title, game_params *params);

将预设游戏配置添加到预设菜单的末尾。

字符串参数 ‘title’ 和游戏参数结构 ‘params’ 本身都必须由调用者动态分配。预设菜单结构将取得其所有权，因此调用者不得释放它。

5.2.4 preset_menu_lookup_by_id()

game_params *preset_menu_lookup_by_id
    (struct preset_menu *menu, int id);

给定一个数字索引，递归搜索预设菜单层次结构以找到对应的菜单项，并返回指向其现有 game_params 结构的指针。

此函数供前端使用（但前端如果愿意以其他方式做事，则不需要使用它）。如果前端发现除了每个菜单项旁边存储数字索引之外存储任何东西都不方便，那么此函数为前端提供了一种简单的方法，以获取用户选择的菜单项对应的实际游戏参数。

5.3 内存分配

Puzzles 对标准内存分配函数有一些中央包装器，它们提供编译时类型检查，以及通过在内存不足时退出应用程序来进行运行时错误检查。这不能提供最好的内存不足恢复，但另一方面，它大大简化了代码的其余部分，因为其他任何地方都不需要担心分配的 NULL 返回。

5.3.1 snew()

var = snew(type);

此宏接受一个参数，即类型名称。它为该类型的一个对象分配空间。如果分配失败，它将调用 fatal() 并且不返回；因此，如果它返回，您可以确信其返回值是非 NULL 的。

返回值被转换为指定的类型，因此编译器将对其与您分配给它的变量进行类型检查。因此，这确保您不会意外分配错误类型大小的内存并将其分配给正确类型的变量（或反之亦然！）。

5.3.2 snewn()

var = snewn(n, type);

此宏是 snew() 的数组形式。它接受两个参数：第一个是数字，第二个是类型名称。它为该类型的许多对象分配空间，并像 snew() 一样返回类型检查的非 NULL 指针。

5.3.3 sresize()

var = sresize(var, n, type);

此宏是 realloc() 的类型检查形式。它接受三个参数：输入内存块、元素的新大小和类型。它将输入内存块调整为足以包含该类型的多个元素的大小。它像 snew() 和 snewn() 一样返回类型检查的非 NULL 指针。

输入内存块可以是 NULL，在这种情况下，此函数的行为将完全像 snewn()。（原则上，任何符合 ANSI 标准的 realloc() 实现都应该能够处理这种情况，但我从未完全信任它在所有地方都能工作。）

5.3.4 sfree()

void sfree(void *p);

此函数几乎等同于 free()。它被提供一个动态分配的块，并释放它。

输入内存块可以是 NULL，在这种情况下，此函数将不执行任何操作。（原则上，任何符合 ANSI 标准的 free() 实现都应该能够处理这种情况，但我从未完全信任它在所有地方都能工作。）

5.3.5 dupstr()

char *dupstr(const char *s);

此函数动态分配 C 字符串的副本。像 snew() 函数一样，它保证返回非 NULL 或根本不返回。

（许多平台提供函数 strdup()。除了保证从不返回 NULL 之外，我的版本的优点是在所有地方都定义，而不是不方便地几乎在所有地方都定义。）

5.3.6 free_cfg()

void free_cfg(config_item *cfg);

此函数正确释放 config_item 数组，包括遍历数组直到末尾并释放每个 u 子联合中预期为动态分配的任何辅助数据项。

（有关 config_item 结构的详细信息，请参见 第 2.3.9 节。）

5.3.7 free_keys()

void free_keys(key_label *keys, int nkeys);

此函数正确释放 key_label 数组，包括每个键的动态分配的标签字符串。

（有关 key_label 结构的详细信息，请参见 第 2.10.7 节。）

5.4 排序和计数树函数

许多游戏需要复杂的算法来生成随机拼图，有些甚至在游戏过程中需要适度复杂的算法。这些算法的一个常见要求是需要维护项目的排序或未排序列表，以便可以方便地删除和添加项目。

对于一般用途，Puzzles 提供了以下一组在内存中维护 2-3-4 树的函数。（2-3-4 树是一种平衡树结构，具有所有查找、插入、删除、拆分和连接都可以在 O(log N) 时间内完成的特性。）

所有这些函数都期望您存储 void * 指针的树。您可以在这些指针中放入任何您喜欢的东西。

通过使用每节点元素计数，这些树结构具有通过它们在树表示的列表中的数字索引查找元素的略微不同寻常的能力。这意味着它们可以用于存储未排序列表（在这种情况下，每次插入新元素时，您必须明确指定要插入的位置）。它们还可以在排序树中进行数字查找，这对于（例如）跟踪变化数据集的中位数可能很有用。

除了存储排序列表外，这些函数还可以用于存储“映射”（关联数组），方法是将树的每个元素定义为（键，值）对。

5.4.1 newtree234()

tree234 *newtree234(cmpfn234 cmp);

创建一个新的空树，并返回指向它的指针。

参数 cmp 确定树上的排序标准。其原型是 

typedef int (*cmpfn234)(void *, void *);

如果您想要排序树，您应该提供一个匹配此原型的函数，该函数返回类似于 strcmp() 的值（如果第一个参数小于第二个参数则为负数，如果大于则为正数，如果相等则为零）。在这种情况下，函数 addpos234() 将不能在您的树上使用（因为所有插入都必须遵守排序顺序）。

如果您想要未排序树，请传递 NULL。在这种情况下，您将无法使用 add234() 或 del234()，或任何其他依赖于排序顺序的函数，如 find234()。您的树将变得更像一个数组，除了它将有效地支持插入和删除以及按数字索引的查找。

5.4.2 freetree234()

void freetree234(tree234 *t);

释放树。此函数不会释放树的元素（因为它们可能不是动态分配的，或者您可能在多个树中存储相同的元素集）；它只会释放树结构本身。如果您想要释放树的所有元素，您应该在将其传递给 freetree234() 之前清空它，方法是使用类似于以下的代码 

while ((element = delpos234(tree, 0)) != NULL)
    sfree(element); /* 或一些更复杂的释放函数 */

5.4.3 add234()

void *add234(tree234 *t, void *e);

将新元素 e 插入树 t 中。此函数期望树是排序的；新元素根据排序顺序插入。

如果树中已经存在与 e 相等的元素，则插入将失败，返回值将是现有元素。否则，插入成功，返回 e。

5.4.4 addpos234()

void *addpos234(tree234 *t, void *e, int index);

将新元素插入未排序树中。由于没有排序顺序来决定新元素的位置，您必须指定要插入的位置。将 index 设置为零将新元素放在列表的开头；将 index 设置为树中当前元素的数量将新元素放在末尾。

返回值是 e，与 add234() 一致（尽管此函数除了内存不足外不会失败，在这种情况下它会崩溃并死亡，而不是返回错误指示）。

5.4.5 index234()

void *index234(tree234 *t, int index);

返回指向树的第 index 个元素的指针，如果 index 超出范围则返回 NULL。树的元素从零开始编号。

5.4.6 find234()

void *find234(tree234 *t, void *e, cmpfn234 cmp);

在排序树中搜索与 e 相等的元素。

如果 cmp 是 NULL，则将使用树的普通比较函数来执行搜索。但是，有时您不希望这样；例如，假设您的每个元素都是一个包含 char * 名称字段的大结构，并且您想要找到具有给定名称的元素。您可以通过构造一个假元素结构，适当设置其名称字段，并将其传递给 find234() 来实现这一点，但您可能会发现将仅名称字符串传递给 find234()，提供一个替代比较函数，该函数期望其参数之一是裸名称，另一个是包含名称字段的大结构，会更方便。

因此，如果 cmp 不是 NULL，则它将用于将 e 与树的元素进行比较。传递给 cmp 的第一个参数将始终是 e；第二个将是树的一个元素。

（有关 cmpfn234 函数指针类型的定义，请参见 第 5.4.1 节。）

返回值是找到的元素，如果搜索不成功则返回 NULL。

5.4.7 findrel234()

void *findrel234(tree234 *t, void *e, cmpfn234 cmp, int relation);

此函数类似于 find234()，但具有执行相对搜索的附加能力。附加参数 relation 可以是以下值之一：

REL234_EQ
仅查找与 e 相等的元素。这与 find234() 的行为完全相同。
REL234_LT
查找严格小于 e 的最大元素。e 可以是 NULL，在这种情况下，它找到整个树中的最大元素（这也可以通过 index234(t, count234(t)-1) 完成）。
REL234_LE
查找小于或等于 e 的最大元素。（也就是说，尽可能找到与 e 相等的元素，但如果失败，则接受比它小的元素。）
REL234_GT
查找严格大于 e 的最小元素。e 可以是 NULL，在这种情况下，它找到整个树中的最小元素（这也可以通过 index234(t, 0) 完成）。
REL234_GE
查找大于或等于 e 的最小元素。（也就是说，尽可能找到与 e 相等的元素，但如果失败，则接受比它大的元素。）

返回值，如前所述，是找到的元素或 NULL（如果没有元素满足搜索条件）。

5.4.8 findpos234()

void *findpos234(tree234 *t, void *e, cmpfn234 cmp, int *index);

此函数类似于 find234()，但具有返回树中找到的元素的索引的附加功能；在成功搜索（非 NULL 返回值）的情况下，该索引被写入 *index。

index 可以是 NULL，在这种情况下，此函数的行为与 find234() 完全相同。

5.4.9 findrelpos234()

void *findrelpos234(tree234 *t, void *e, cmpfn234 cmp, int relation,
                    int *index);

此函数结合了 findrel234() 和 findpos234() 的所有功能。

5.4.10 del234()

void *del234(tree234 *t, void *e);

在树中找到与 e 相等的元素，删除它，并返回它。

输入树必须是排序的。

找到的元素可能是 e 本身，也可能只是与它相等。

如果未找到这样的元素，返回值是 NULL。

5.4.11 delpos234()

void *delpos234(tree234 *t, int index);

删除树中位置 index 的元素，并返回它。

如果索引超出范围，返回值是 NULL。

5.4.12 count234()

int count234(tree234 *t);

返回树中当前元素的数量。

5.4.13 splitpos234()

tree234 *splitpos234(tree234 *t, int index, bool before);

在给定位置将输入树拆分为两部分，并创建一个新树，包含该位置一侧的所有元素。

如果 before 为 true，则位置 index 或之后的所有项目都留在输入树中，该点之前的项目在新树中返回。否则，情况相反：位置 index 或之后的所有项目都移动到新树中，之前的项目留在旧树中。

如果 index 等于 0 或输入树中的元素数量，则两棵树中的一棵将最终为空（这不是错误条件）。如果 index 在任一方向上进一步超出范围，操作将完全失败并返回 NULL。

此操作在 O(log N) 时间内完成，无论树有多大或拆分如何平衡或不平衡。

5.4.14 split234()

tree234 *split234(tree234 *t, void *e, cmpfn234 cmp, int rel);

根据排序顺序拆分排序树。

rel 可以是 第 5.4.7 节 中描述的任何关系常量，除了 REL234_EQ。具有与 e 该关系的所有元素将被转移到新树中；其余元素将留在旧树中。

参数 cmp 与 find234() 中的语义相同：如果它不是 NULL，则在将元素与 e 进行比较时，它将替代树自己的比较函数，使得 e 本身始终是其两个操作数中的第一个。

同样，此操作在 O(log N) 时间内完成，无论树有多大或拆分如何平衡或不平衡。

5.4.15 join234()

tree234 *join234(tree234 *t1, tree234 *t2);

通过连接它们表示的列表将两棵树连接在一起。t2 的所有元素都被移动到 t1 中，使得它们出现在 t1 的元素之后。树 t2 被释放；返回值是 t1。

如果您将此函数应用于排序树并且它违反了排序顺序（即 t2 中的最小元素小于或等于 t1 中的最大元素），操作将失败并返回 NULL。

此操作在 O(log N) 时间内完成，无论要连接的树有多大。

5.4.16 join234r()

tree234 *join234r(tree234 *t1, tree234 *t2);

以与 join234() 完全相同的方式连接两棵树，但这次组合树在 t2 中返回，而 t1 被销毁。t1 中的元素仍然出现在 t2 中的元素之前。

同样，此操作在 O(log N) 时间内完成，无论要连接的树有多大。

5.4.17 copytree234()

tree234 *copytree234(tree234 *t, copyfn234 copyfn,
                     void *copyfnstate);

复制整个树。

如果 copyfn 是 NULL，树将被复制，但元素不会被复制；即，新树将包含指向与旧树完全相同的物理元素的指针。

如果您想在操作期间复制每个实际元素，您可以改为在 copyfn 中传递一个复制每个元素的函数。该函数具有原型 

typedef void *(*copyfn234)(void *state, void *element);

每次调用它时，state 参数将设置为您作为 copyfnstate 传递的值。

5.5 不相交集森林

本节描述了一组实现数据结构的函数，该数据结构被称为“并查集”或“Tarjan 的不相交集森林”。在这个代码库中，它被普遍缩写为“dsf”。

dsf 表示一组元素，这些元素被划分为“等价类”，在等价关系被增量添加的情况下。也就是说，所有元素开始时被认为是不同的，您通过 dsf_merge() 操作逐渐声明越来越多的元素相等，该操作表示从现在开始，两个特定元素应该被视为相等。

例如，如果我开始时 A,B,U,V 都不同，并且我将 A 与 B 合并，将 U 与 V 合并，那么该结构将告诉我 A 和 U 不是等价的。但如果我然后将 B 与 V 合并，那么之后，该结构将告诉我 A 和 U 是 等价的，通过跟随它知道的等价关系的传递链。

因此，dsf 数据结构非常适合跟踪无向图中的增量连接性（同样，“增量”意味着您只添加边，从不删除它们），以及其他您逐渐获得关于什么与什么相同的知识的应用程序。它广泛用于拼图求解器和生成器算法，有时也用于游戏过程中。

dsf 操作的时间复杂度在理论上不是完全恒定时间，但它非常接近，以至于在实践中没有区别。特别是，每当 dsf 必须做非平凡的工作时，它会更新结构，这样就不需要第二次做这项工作。使用 dsf 操作，无需担心它们需要多长时间！

对于一些拼图游戏应用，通过额外信息增强此数据结构以了解等价类的元素如何相互关联是有用的。您可能有多种方法来做到这一点；这里支持的一种方法在您要跟踪的对象最终会处于两种状态之一（例如，黑/白或开/关）的情况下很有用，对于任何两个对象，您要么知道它们处于相同的状态，要么知道它们处于相反的状态，要么还不知道。Puzzles 称这为“翻转 dsf”：它跟踪同一等价类中的对象是否相对于彼此翻转。

除了查询两个元素是否等价外，此 dsf 实现还允许您询问给定等价类中的元素数量，以及类中的最小元素。（后者用于例如决定在 Keen 拼图的每个区域中打印线索的方块。）

5.5.1 dsf_new(), dsf_new_flip(), dsf_new_min()

DSF *dsf_new(int size);
DSF *dsf_new_flip(int size);
DSF *dsf_new_min(int size);

这些函数中的每一个都为描述 size 个元素的 dsf 分配空间，并将其初始化，使得每个元素都在自己的等价类中。

dsf 描述的元素由从 0 到 size-1 的整数表示。

dsf_new_flip() 将创建一个 dsf，该 dsf 具有跟踪同一等价类中的对象是否相对于彼此翻转的额外能力。

dsf_new_min() 将创建一个 dsf，该 dsf 具有跟踪每个等价类的最小元素的额外能力。

任何这些函数的返回对象都必须使用 dsf_free() 释放。

5.5.2 dsf_free()

void dsf_free(DSF *dsf);

释放由任何 dsf_new() 函数分配的 dsf。

5.5.3 dsf_reinit()

void dsf_reinit(DSF *dsf);

将现有 dsf 重新初始化为所有元素都不同的状态，而无需释放和重新分配它。

5.5.4 dsf_copy()

void dsf_copy(DSF *to, DSF *from);

将一个 dsf 的内容复制到另一个 dsf 上。之前存储在 to 中的所有内容都被覆盖。

两个 dsf 必须以相同的大小创建，并且目标 dsf 不得具有源 dsf 没有的任何额外信息。

5.5.5 dsf_merge()

void dsf_merge(DSF *dsf, int v1, int v2);

更新 dsf，使得元素 v1 和 v2 现在将被视为在同一等价类中。如果它们已经在同一类中，此函数将安全地不执行任何操作。

此函数不得在翻转 dsf 上调用。请改用 dsf_merge_flip。

5.5.6 dsf_canonify()

int dsf_canonify(DSF *dsf, int val);

返回 dsf 中包含 val 的等价类的“规范”元素。这将是同一等价类的某个元素。因此，为了确定两个元素是否在同一等价类中，您可以对它们都调用 dsf_canonify，并比较结果。

如果 dsf 通过 dsf_merge 发生变异，规范元素不一定保持相同。但在两次 dsf_merge 调用之间，它们保持相同。

5.5.7 dsf_size()

int dsf_size(DSF *dsf, int val);

返回当前包含 val 的等价类中的元素数量。

val 本身计数，因此在新创建的 dsf 中，返回值将是 1。

5.5.8 dsf_merge_flip()

void edsf_merge(DSF *dsf, int v1, int v2, bool flip);

更新翻转 dsf，使得元素 v1 和 v2 在同一等价类中。如果 flip 为 false，它们将被视为彼此处于相同状态；如果 flip 为 true，则它们将被视为处于相反状态。

如果 v1 和 v2 已经在同一等价类中，则新的 flip 值将与 edsf 之前的信念进行检查，如果您与之矛盾，将发生断言失败。

例如，如果您从空白翻转 dsf 开始并执行以下操作： 

dsf_merge_flip(dsf, 0, 1, false);
dsf_merge_flip(dsf, 1, 2, true);

然后它将创建一个 dsf，其中元素 0,1,2 都在同一类中，0,1 彼此处于相同状态，2 与两者处于相反状态。然后此调用将不执行任何操作，因为它与 dsf 已经知道的一致： 

dsf_merge_flip(dsf, 0, 2, true);

但此调用将失败断言： 

dsf_merge_flip(dsf, 0, 2, false);

5.5.9 dsf_canonify_flip()

int dsf_canonify_flip(DSF *dsf, int val, bool *inverse);

像 dsf_canonify() 一样，这返回包含 val 的 dsf 等价类的规范元素。

然而，它只能在翻转 dsf 上调用，并且它还使用一个标志填充 *flip，该标志指示 val 和规范元素是否处于相反状态：如果它们处于相反状态，则为 true，如果它们处于相同状态，则为 false。

因此，如果您想知道 v1 和 v2 之间的关系，您可以这样做： 

bool inv1, inv2;
int canon1 = dsf_canonify_flip(dsf, v1, &inv1);
int canon2 = dsf_canonify_flip(dsf, v2, &inv2);
if (canon1 != canon2) {
    // v1 和 v2 没有已知关系
} else if (inv1 == inv2) {
    // v1 和 v2 已知彼此处于相同状态
} else {
    // v1 和 v2 已知处于相反状态
}

5.5.10 dsf_minimal()

int dsf_minimal(DSF *dsf, int val);

返回 dsf 中包含 val 的等价类的最小元素。

为了使此函数工作，dsf 必须使用 dsf_new_min() 创建。

5.6 待办队列

本节描述了一组实现“待办队列”的函数，这是一种简单的去重待办列表机制。代码称这为“tdq”。

tdq 可以存储高达给定大小的整数（在创建时指定）。但它不能存储相同的整数多次。因此，您可以快速确保整数在队列中（如果它已经在那里，将不执行任何操作），并且您可以快速从队列中弹出整数并返回它，两者都在恒定时间内。

这个想法是，您可能在游戏求解器中使用它，在那种游戏中，更新您对网格中一个方块的知识意味着有一组特定的其他方块（如其邻居），现在值得尝试进一步的推理。因此，您保留一个待办队列，其中包含您计划下一步查看的所有网格方块，每次在一个方块中进行推理时，您将相邻的方块添加到待办队列中，以确保它们在那之后再次被查看。

在推理大多是局部的求解器中，这避免了每次通过遍历整个网格来找到下一个要做的事情的减慢：相反，您可以不断检查 tdq 以获取特定要查看的方块，直到用完。

然而，常见的情况是，游戏中大多数推理是局部的，但并非全部。在这种情况下，当您的 tdq 为空时，您可以使用 tdq_fill() 用网格中的每个方块重新填充它，这将强制再次迭代所有内容。然后，如果 tdq 在您没有取得任何进展的情况下再次为空，就放弃。

5.6.1 tdq_new()

tdq *tdq_new(int n);

为跟踪从 0 到 size-1 范围内的项目的 tdq 分配空间。

5.6.2 tdq_free()

void tdq_free(tdq *tdq);

释放 tdq。

5.6.3 tdq_add()

void tdq_add(tdq *tdq, int k);

将值 k 添加到 tdq。如果 k 已经在待办列表中，则不执行任何操作。

5.6.4 tdq_remove()

int tdq_remove(tdq *tdq);

从 tdq 中删除一个项目并返回它。如果 tdq 为空，则返回 -1。

5.6.5 tdq_fill()

void tdq_fill(tdq *tdq);

用它可以跟踪的每个元素填充 tdq。

5.7 在图和网格中查找环路

许多在网格或图上玩的拼图都有一个共同的游戏元素，即将事物连接成路径，以避免形成环路（或者，也许，避免形成错误类型的环路）。

仅确定图中是否存在环路很容易，使用跟踪顶点之间连接性的 dsf。只需遍历图的每条边，合并每条边两端的两个顶点——但在这样做之前，检查这些顶点是否已经已知彼此连接，如果是，则新边即将完成一个环路。

但是，如果您还想精确识别任何环路的边集，例如在游戏过程中将整个环路标记为红色，那么这是一个更难的问题。此 API 在此处提供，供所有拼图用于此目的。

5.7.1 findloop_new_state()

struct findloopstate *findloop_new_state(int nvertices);

为 findloop 算法分配新的状态结构，能够处理最多 nvertices 个顶点的图。顶点将由从 0 到 nvertices-1 的整数表示。

5.7.2 findloop_free_state()

void findloop_free_state(struct findloopstate *state);

释放由 findloop_new_state() 分配的状态结构。

5.7.3 findloop_run()

bool findloop_run(struct findloopstate *state, int nvertices,
                  neighbour_fn_t neighbour, void *ctx);

运行环路查找算法，该算法将探索图并识别每条边是否是任何环路的一部分。

该算法将调用提供的函数 neighbour 来列出每个顶点的相邻顶点。它应该具有此原型： 

int neighbour(int vertex, void *ctx);

在此回调中，当算法首次为给定顶点调用它时，vertex 将是顶点的索引。该函数应返回该顶点的一个邻居的索引，如果没有则返回负数。

如果函数返回顶点，算法将再次调用 neighbour，这次 vertex 参数为负数，这意味着“请给我上次的同一顶点的另一个邻居”。同样，该函数应返回顶点索引，或返回负数表示没有更多顶点。

传递给 findloop_run() 的 ctx 参数被原样传递给 neighbour，因此您可以将其指向您的游戏状态或求解器状态或其他任何东西。

返回值为 true 如果图中至少存在一个环路，false 如果不存在环路。此外，算法状态将填充以下查询函数可以用来询问单个图边的信息。

5.7.4 findloop_is_loop_edge()

bool findloop_is_loop_edge(struct findloopstate *state,
                           int u, int v);

查询顶点 u 和 v 之间的图边是否是环路的一部分。如果是，返回值为 true，否则为 false。

5.7.5 findloop_is_bridge()

bool findloop_is_bridge(struct findloopstate *pv,
    int u, int v, int *u_vertices, int *v_vertices);

查询顶点 u 和 v 之间的图边是否是“桥”，即如果移除它会将图分解为（更多）断开的组件的边。

这是“环路边”标准的精确 inverse：顶点从 findloop_is_loop_edge() 返回 true 当且仅当它从 findloop_is_bridge() 返回 false，反之亦然。

然而，findloop_is_bridge() 返回更多信息。如果它返回 true，那么它还会用桥的 u 和 v 侧连接的顶点数量填充 *u_vertices 和 *v_vertices。

例如，如果你有三个顶点 A,B,C 彼此连接，四个顶点 U,V,W,X 彼此连接，以及 A 和 V 之间的一条边，那么对 A,V 对调用 findloop_is_bridge() 将返回 true（移除该边会将两组分开），并报告 A 侧有三个顶点，V 侧有四个顶点。

5.8 从 n 个元素中选择 r 个

本节描述了一个小 API，用于迭代所有从 n 个元素中选择 r 个的组合。

例如，如果你要求所有从 5 个元素中选择 3 个的组合，你会得到集合 {0,1,2}、{0,1,3}、{0,1,4}、{0,2,3}、{0,2,4}、{0,3,4}、{1,2,3}、{1,2,4}、{1,3,4} 和 {2,3,4}。

这些函数使用一个称为 combi_ctx 的结构，其中包含一个元素 int *a 保存每个返回的组合，以及其他仅用于实现的字段。

5.8.1 new_combi()

combi_ctx *new_combi(int r, int n);

为枚举从 n 个元素中选择 r 个的组合分配新的 combi_ctx 结构。

5.8.2 free_combi()

void free_combi(combi_ctx *combi);

释放 combi_ctx 结构。

5.8.3 reset_combi()

void reset_combi(combi_ctx *combi);

将现有的 combi_ctx 结构重置到其迭代的开始

5.8.4 next_combi()

combi_ctx *next_combi(combi_ctx *combi);

从 combi_ctx 请求组合。

如果没有更多要返回的组合，返回值为 NULL。否则，它返回输入结构 combi，表示它已经填充了 combi->a[0]、combi->a[1]、...、combi->a[r-1]，其中包含从 0 到 n-1 的递增序列的不同整数。

5.9 其他实用工具函数和宏

本节包含所有未合理归入其他地方的实用工具函数。

5.9.1 max() 和 min()

Puzzles 主头文件定义了相当标准的宏 max() 和 min()，每个宏都接受两个参数并返回分别比较大或小的那个。

这些宏可能会多次评估它们的参数。避免副作用。

5.9.2 MAX_DIGITS()

在 Puzzles 主头文件中定义的 MAX_DIGITS() 宏接受一个类型（或该类型的变量），并展开为一个整数常量，表示该类型的数字在使用 printf() 的 %u 或 %d 格式格式化为十进制数时可能展开的字符数的合理上限。这对于分配一个保证足够大的固定大小缓冲区以 sprintf() 值非常有用。不要忘记为尾随的 '\0' 添加一个！

5.9.3 PI

Puzzles 主头文件定义了一个宏 PI，它展开为表示 pi 的浮点常量。

（我一直不明白为什么 ANSI 的 <math.h> 不定义这个。它会非常有用！）

5.9.4 obfuscate_bitmap()

void obfuscate_bitmap(unsigned char *bmp, int bits, bool decode);

此函数通过加密方法模糊数据的内容。它对隐藏信息的游戏（如 Mines、Guess 或 Black Box）很有用，其中游戏 ID 理论上揭示了玩家应该尝试猜测的所有信息。因此，为了让玩家能够相互发送游戏 ID 而不会通过查看它们意外破坏结果游戏，这些游戏使用此函数混淆它们的游戏 ID。

虽然混淆函数是加密的，但由于它没有密钥，因此不能正确称为加密。因此，任何有足够动力的人都可以重新实现它，或从 Puzzles 源代码中破解它，并去除这些游戏 ID 的混淆以查看其下的内容。（实际上，他们通常可以通过将游戏 ID 输入到自己的拼图副本中并点击 Solve 来更轻松地做到这一点。）目标不是防止坚定的攻击者；目标只是保护想要诚实地玩游戏的人意外破坏自己的乐趣。

输入参数 bmp 指向要混淆的内存块。bits 给出数据的长度。请注意，该长度以位而不是字节为单位：如果您要求混淆部分字节，您将得到它。字节被认为是从上到下使用的：因此，例如，将 bits 设置为 10 将覆盖整个 bmp[0] 和 bmp[1] 的前两位。部分使用的字节的剩余部分是未定义的（即，它可能被函数损坏）。

参数 decode 对于编码操作是 false，对于解码操作是 true。每个都是另一个的逆。（如果您真的想要，您可以通过解码来混淆并通过编码来恢复明文；它仍然应该工作。）

输入位图是就地处理的。

5.9.5 bin2hex()

char *bin2hex(const unsigned char *in, int inlen);

此函数接受输入字节数组并将其转换为 ASCII 字符串，以（小写）十六进制编码这些字节。它返回包含该编码的动态分配的字符串。

此函数对于将 obfuscate_bitmap() 的结果编码为可打印的 ASCII 以用于游戏 ID 很有用。

5.9.6 hex2bin()

unsigned char *hex2bin(const char *in, int outlen);

此函数接受包含十六进制数字的 ASCII 字符串，并将其转换回长度为 outlen 的字节数组。如果字符串中没有足够的十六进制数字，结果数组的内容将是未定义的。

此函数是 bin2hex() 的逆函数。

5.9.7 fgetline()

char *fgetline(FILE *fp);

此函数从标准 C 输入流读取单行文本，并将其作为动态分配的字符串返回。返回的字符串末尾仍然有一个换行符。

5.9.8 arraysort()

排序数组，比标准 C qsort() 稍微灵活一些。

此函数实际上是作为宏实现的，因此它没有这样的原型。但您可以想象它有这样的原型： 

void arraysort(element_t *array, size_t nmemb,
               arraysort_cmpfn_t cmp, void *ctx);

其中 element_t 是未指定的类型。

（实际上，有一个底层函数，它接受一个额外的参数，给出每个数组元素的大小。但调用者被鼓励使用此宏版本，它使用 sizeof 自动填充。）

此函数的行为基本上类似于 qsort()：它期望 array 指向 nmemb 元素的数组，并且它将按照比较函数 cmp 指定的顺序对它们进行就地排序。

比较函数应该具有此原型： 

int cmp(const void *a, const void *b, void *ctx);

其中 a 和 b 指向要比较的两个元素，返回值为负如果 a<b（即，a 应该在输出数组中出现在 b 之前），正如果 a>b，或零如果 a=b。

传递给 arraysort() 的 ctx 参数直接传递给比较函数。这是使 arraysort() 比标准 qsort() 更灵活的特性：它允许您以动态方式改变排序标准，而无需在调用者中为比较函数读取的全局变量编写代码。

5.9.9 colour_mix()

void colour_mix(const float src1[3], const float src2[3], float p,
                float dst[3]);

此函数以指定的比例混合颜色 src1 和 src2，产生 dst。p 是结果中 src2 的比例。因此，如果 p 为 1.0，dst 将与 src2 相同。如果 p 为 0.0，dst 将与 src1 相同。如果 p 在两者之间，dst 也会在两者之间。p 不限于 0.0 到 1.0 的范围。该范围之外的值将产生外推颜色，这对于某些目的可能有用，但也可能产生不可能的颜色。

5.9.10 game_mkhighlight()

void game_mkhighlight(frontend *fe, float *ret,
                      int background, int highlight, int lowlight);

拼图游戏的图形使用高光和低光来指示“凸起”或“凹陷”部分是相当常见的。Fifteen、Sixteen 和 Twiddle 是很好的例子。

使用这种图形风格的拼图存在风险，如果它们只是使用前端提供的任何背景颜色，因为该背景颜色可能太亮或太暗，无法看到任何高光。（特别是，前端指定默认背景颜色为白色并不少见。）

因此，这样的拼图可以从它们的 colours() 例程（第 2.8.6 节）调用此实用函数。您传递给它前端句柄、指向返回数组开头的指针以及三个颜色索引。它将：

  • 调用 frontend_default_colour()（第 4.45 节）获取前端的默认背景颜色
  • 如果不合适，改变该颜色的亮度
  • 定义该颜色的更亮和更暗的变体，用作高光和低光
  • 将这些结果写入 ret 数组的相关位置。

因此，ret[background*3] 到 ret[background*3+2] 将设置为定义合理背景颜色的 RGB 值，类似地，highlight 和 lowlight 将设置为合理的颜色。

highlight 或 lowlight 可以作为 -1 传递，表示后端不需要高光或低光颜色。

5.9.11 game_mkhighlight_specific()

void game_mkhighlight_specific(frontend *fe, float *ret,
    int background, int highlight, int lowlight);

此函数的行为与 game_mkhighlight() 完全相同，除了它期望背景颜色已经在元素 ret[background*3] 到 ret[background*3+2] 中填充。它将填充其他两种颜色作为该颜色的更亮和更暗的版本。

如果您想在多个基本颜色中显示拼图的浮雕部分，这很有用。

5.9.12 button2label()

char *button2label(int button);

此函数为 button 的某些通用值生成描述性文本标签，这些值是可以传递给中端的按钮代码。

button 的允许值是 A-Z, a-z, 0-9, CURSOR_{UP,DOWN,LEFT,RIGHT,SELECT} 和 '\b'。不允许其他值。传递不支持的值将产生致命错误。

此函数的预期用途是生成无键盘前端可以向用户呈现为可以发送到后端的键的字符串。例如，CURSOR_UP 产生 "Up"。大写和小写字母数字字符产生包含该字符的单例字符串。退格字符 '\b' 产生 "Clear"。

此函数应仅在后端的 request_keys()（第 2.10.7 节）函数返回的 key_label 项的 label 字段设置为 NULL 时调用；在这种情况下，可以将相应的 button 字段传递给此函数以获得适当的通用标签。但是，如果 label 字段不是 NULL，则不应使用此函数。

返回的字符串是动态分配的，应由调用者 sfree。

5.9.13 move_cursor()

char *move_cursor(int button, int *x, int *y, int w, int h,
                  bool wrap, bool *visible);

此函数可以由 interpret_move() 调用来实现用于在网格周围移动光标的默认键盘 API。

button 是传递给 interpret_move() 的相同值。如果它不是 CURSOR_UP、CURSOR_DOWN、CURSOR_LEFT 或 CURSOR_RIGHT 中的任何一个，函数将不执行任何操作。

x 和 y 指向两个整数，输入时给出光标在正方形网格中的当前位置。w 和 h 给出网格的尺寸。返回时，x 和 y 根据按下的箭头键更新为光标的新位置。

此函数假设网格坐标从 0 到 w-1（从左到右），以及从 0 到 h-1（从上到下）。

如果 wrap 为 true，则尝试将光标移出网格的任何边缘将导致它环绕到相对边缘的相应正方形。如果 wrap 为 false，则此类移动将无效。

如果 visible 不是 NULL，它指向一个指示光标是否可见的标志。如果 button 表示光标移动键，这将设置为 true。

函数返回可以由 interpret_move() 返回的特殊常量之一。如果 button 未被识别，返回值为 MOVE_UNUSED；如果 x、y 或 visible 已更新，返回值为 MOVE_UI_UPDATE；否则返回 MOVE_NO EFFECT。

5.9.14 divvy_rectangle()

int *divvy_rectangle(int w, int h, int k, random_state *rs);

发明一个矩形的随机分割成相同大小的多格骨牌，例如在拼图模式下的 Solo 拼图的块布局中，或 Palisade 拼图的解决方案中找到的。

w 和 h 是矩形的尺寸。k 是所需的多格骨牌的大小。它必须是 w*h 的因数。

rs 是用于选择矩形的随机分割的随机数的 random_state。

返回值是一个 dsf（参见 第 5.5 节），其等价类对应于矩形被分割成的多格骨牌。dsf 的索引形式为 y*w+x，对于坐标为 x,y 的单元格。

5.9.15 domino_layout()

int *domino_layout(int w, int h, random_state *rs);

发明一个用多米诺骨牌随机平铺的矩形。

w 和 h 是矩形的尺寸。如果它们都是奇数，则会留下一个方块未平铺。

rs 是用于选择矩形的随机分割的随机数的 random_state。

返回值是一个数组，其中元素 y*w+x 表示坐标为 x,y 的单元格。数组的每个元素给出其多米诺骨牌另一端的索引（以相同的表示形式）。如果有剩余的方块，则该元素包含其自己的索引。

5.9.16 domino_layout_prealloc()

void domino_layout_prealloc(int w, int h, random_state *rs,
                            int *grid, int *grid2, int *list);

就像 domino_layout()，但不进行内存分配。如果您期望需要生成相同网格的许多多米诺骨牌平铺，您可以使用此函数来节省分配器开销。

grid 和 grid2 各自应该有空间用于 w*h 个整数。list 应该有空间用于 2*w*h 个整数。

返回的数组在 grid 中交付。

5.9.17 STRIP_BUTTON_MODIFIERS()

此宏在 Puzzles 主头文件中定义，从作为参数传递的键代码中剥离修饰符标志。它等效于与 ~MOD_MASK 进行按位与操作。

5.9.18 swap_regions()

void swap_regions(void *av, void *bv, size_t size);

交换两个大小为 size 字节的内存区域。两个区域不得重叠。

5.9.19 compare_integers()

int compare_integers(const void *av, const void *bv);

比较 av 和 bv 指向的整数。如果第一个参数分别小于、等于或大于第二个参数，它返回小于、等于或大于零的整数。此函数旨在传递给 qsort() 以按升序排序整数。

5.9.20 GET_HANDLE_AS_TYPE()

#define GET_HANDLE_AS_TYPE(dr, type) ((type*)((dr)->handle))

此宏在 Puzzles 主头文件中定义，从 drawing * 指针中检索 handle 字段并将其转换为 type *。它旨在用于绘图 API 函数的实现，其中此操作是惯用的。

5.9.21 getenv_bool()

bool getenv_bool(const char *name, bool dflt)

这是标准库函数 getenv() 的简单包装器。它查询名为 name 的环境变量，如果变量存在并且看起来有点真实（任何以 y、Y、t 或 T 开头的值），则返回 true。如果变量定义为其他值，则返回 false，如果未定义，则函数返回 dflt。这可用于实现对于标准首选项机制来说太晦涩的首选项（参见 第 2.3.11 节）。
