第 3 章：网络

（注意：这个游戏的 Windows 版本被称为 NETGAME.EXE，以避免与 Windows 自己的 NET.EXE 冲突。）

我最初是在一个名为 FreeNet [1] 的 Flash 游戏中看到这个的，由 Pavils Jurjans 编写；还有几个其他的实现，名称为 NetWalk。计算机通过连接网格中方格的中心来准备一个网络，然后通过随机旋转每个瓦片来打乱网络。你的工作是将其全部旋转回原位。成功的解决方案将是一个完全连接的网络，没有闭合回路。作为视觉辅助，所有连接到中间那个的瓦片都会被高亮显示。

[1] http://www.jurjans.lv/stuff/net/FreeNet.htm

3.1 Net 控制

这个游戏可以用键盘或鼠标玩。控制如下：

选择瓦片：鼠标指针，箭头键
逆时针旋转瓦片：鼠标左键，'A' 键
顺时针旋转瓦片：鼠标右键，'D' 键
旋转瓦片 180 度：'F' 键
锁定（或解锁）瓦片：鼠标中键，Shift+点击，'S' 键
一旦你确定了瓦片的方向，你可以锁定它。你也可以再次解锁它，但在锁定时你不能意外地转动它。

以下控制对于完成游戏不是必需的，但可能有用：

移动网格：Shift + 箭头键
在环绕的网格上，你可以移动网格的原点，这样在网格相反两侧的瓦片可以一起看到。
移动中心：Ctrl + 箭头键
你可以更改用作高亮显示源的瓦片。（这最终无关紧要，因为在正确的解决方案中，每个瓦片都会连接到其他每个瓦片，但在解决拼图的中间阶段可能会有所帮助。）
打乱瓦片：'J' 键
这个键将所有未锁定的瓦片转到随机方向。

（第 2.1 节 中描述的所有操作也可用。）

3.2 Net 参数

这些参数可从“类型”菜单的“自定义...”选项获得。

宽度，高度
网格的大小（以瓦片为单位）。
墙壁环绕
如果选中，流可以从左边缘流到右边缘，从顶部流到底部，反之亦然。
障碍概率
一个介于 0.0 和 1.0 之间的数字，控制是否在两个瓦片之间放置不可移动的障碍以防止它们之间的流动（数字越高，障碍越多）。由于障碍是不可移动的，它们作为解决方案的约束（即提示）。

网络中的网格生成已经精心安排，使得障碍与网格的其余部分无关。这意味着如果你记下用于生成当前拼图的随机种子（见 第 2.2 节），更改 障碍概率 参数，然后重新输入相同的随机种子，你应该看到完全相同的起始网格，唯一的变化是障碍的数量。所以如果你在特定的网格上卡住了，需要提示，你可以启动另一个网络实例，设置相同的参数但更高的障碍概率，然后输入原始网络窗口中的游戏种子。

确保唯一解决方案
通常，网络会确保它呈现的拼图只有一个解决方案。具有歧义部分的拼图可能更困难和更微妙，所以如果你愿意，你可以关闭这个功能，冒险获得有歧义的拼图。（此外，找到所有可能的解决方案对高级玩家来说可能是一个额外的挑战。）

3.3 网络用户首选项

在支持用户首选项的平台上，“游戏”菜单上的“首选项”选项将允许你配置何时将回路高亮显示为错误。默认情况下，它们总是被高亮显示；通过更改此选项，你可以要求仅当形成回路的每个瓦片都被锁定时才高亮显示回路。这避免了回路高亮显示作为剧透，用于你甚至还没有查看的方块的可用推断。
