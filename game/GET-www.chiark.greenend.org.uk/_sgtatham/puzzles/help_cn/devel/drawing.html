第3章：绘图 API

后端函数 redraw()（第2.8.11节）需要在窗口的绘图区域上绘制拼图的图形。后端函数 print() 类似地在纸上绘制拼图（如果拼图可打印）。为了可移植地执行此操作，后端提供了一个绘图 API，允许它直接与前端通信。在本章中，我记录了这个 API，既为了尝试使用它的后端作者，也为了尝试实现它的前端作者。

后端看到的绘图 API 是一组全局函数的集合，每个函数都接受一个指向 drawing 结构（“绘图对象”）的指针。这些对象作为参数提供给后端的 redraw() 和 print() 函数。

实际上，这些全局函数不是由前端直接实现的；相反，它们在 drawing.c 中集中实现，形成一个小的中间件。前端提供的绘图 API 是一个包含一组函数指针的结构，加上一个“void *”句柄，该句柄作为后端使用的同一个绘图对象的成员间接传递给这些函数中的每一个。这使得单个前端能够在必要时在绘图 API 的多个实现之间切换。例如，Windows API 提供了一个集成到处理窗口绘图的同一个 GDI 中的打印机制，因此同一个 API 实现可以处理绘图和打印；但在 Unix 上，应用程序打印的最常见方式是直接生成 PostScript 输出，尽管编写一个单一的（例如）draw_rect() 函数是可能的，该函数检查全局标志以决定是执行 GTK 绘图操作还是向文件输出 PostScript，但拥有两个单独的函数并在适当的时候在它们之间切换会好得多。

绘图时，拼图窗口由像素坐标索引，左上角像素定义为 (0,0)，右下角像素定义为 (w-1,h-1)，其中 w 和 h 是后端函数 compute_size()（第2.8.4节）返回的宽度和高度值。

坐标系使用的像素可能与显示器的物理像素大小不同。在高分辨率显示器上，绘图 API 中的一个像素可能对应于多个物理像素。然而，保证 API 像素可以被准确覆盖而不影响其相邻像素，因此一般来说，API 像素将对应于设备像素的整数边矩形。除非另有说明，本文档中的“像素”一词指的是绘图 API 的逻辑像素，而不一定是显示设备的物理像素。

打印时，拼图的打印区域以完全相同的方式索引（由打印模块 printing.c 提供任意的瓦片大小），以方便在绘图和打印例程之间共享代码。然而，打印时，拼图可能不再假设坐标单位与像素有任何关系；打印机的实际分辨率在打印时可能甚至不知道，因此坐标单位可能比像素小或大。拼图的打印函数应该限制自己绘制几何形状，而不是繁琐的像素操作。

拼图的重绘函数可以假设它们绘制的表面是持久的。每个前端都有责任在面对 GUI 窗口暴露问题等情况时保留拼图的窗口内容。不允许请求后端重绘它已经绘制的窗口的任何部分，除非由于在拼图中进行移动而实际发生了变化。

大多数前端通过让绘图例程在存储的位图上绘制而不是直接在窗口上绘制，并在每次需要重绘窗口的一部分时将位图复制到窗口来实现这一点。因此，后端每次进行绘图时，必须通知前端它访问了窗口的哪些部分，从而哪些部分需要重绘，这一点至关重要。这是通过调用 draw_update()（第3.1.12节）来完成的。

旧绘图的持久性是方便的。然而，拼图在如何更新其绘图区域方面应该非常小心。问题是一些前端执行抗锯齿绘图：抗锯齿绘图函数不是简单地在保持每个像素不变或用指定颜色绘制它之间进行选择，而是会根据图形占用像素的比例（可能由一些启发式调整因子修改）在图形边界的像素中混合原始颜色和新颜色。所有这些都会为曲线和对角线产生更平滑的外观。

反复绘制抗锯齿图形的一个不幸效果是，图形边界周围的像素会逐渐被“墨水”饱和，边界似乎会“扩散”。更糟糕的是，用不同颜色重绘图形不会完全覆盖旧的边界像素，因此最终结果是相当丑陋的污迹。

避免令人不愉快的抗锯齿伪影的好策略是识别需要重绘的多个矩形区域，将它们清除为背景颜色，然后从头开始重绘它们的内容，同时小心不要超出原始矩形的边界。clip() 函数（第3.1.10节）在这里非常有用。基于方格网格的游戏通常可以相当容易地做到这一点。其他游戏可能需要更加小心。例如，Loopy 的重绘函数首先识别需要更新的显示部分。然后，如果更改相当局部化，它会清除并重新绘制包含每个更改区域的矩形。否则，它会放弃并从头开始重绘整个网格。

如果非常小心使用的绘图函数，就可以避免清除背景并从头开始重绘：如果一个函数被记录为在某些情况下不进行抗锯齿，你可以依赖绘图中的每个像素要么完全保持不变，要么被设置为请求的颜色，而不执行任何混合。

在以下各节中，我首先讨论后端看到的绘图 API，然后讨论前端看到的几乎相同的函数指针形式。

3.1 后端看到的绘图 API

本节记录后端绘图 API，形式为接受 drawing 对象作为参数的函数。

3.1.1 draw_rect()

void draw_rect(drawing *dr, int x, int y, int w, int h,
               int colour);

在拼图窗口中绘制一个填充的矩形。

x 和 y 给出矩形左上角像素的坐标。w 和 h 给出其宽度和高度。因此，矩形的水平范围从 x 到 x+w-1（含），垂直范围从 y 到 y+h-1（含）。

colour 是后端函数 colours()（第2.8.6节）返回的颜色数组的整数索引。

没有单独的像素绘制函数。如果你想绘制单个像素，推荐的方法是使用 draw_rect()，宽度和高度设置为 1。

与许多其他绘图函数不同，此函数保证是像素完美的：矩形将被清晰定义，不会被抗锯齿或类似处理。

此函数可用于绘图和打印。

3.1.2 draw_rect_outline()

void draw_rect_outline(drawing *dr, int x, int y, int w, int h,
                       int colour);

在拼图窗口中绘制一个轮廓矩形。

x 和 y 给出矩形左上角像素的坐标。w 和 h 给出其宽度和高度。因此，矩形的水平范围从 x 到 x+w-1（含），垂直范围从 y 到 y+h-1（含）。

colour 是后端函数 colours()（第2.8.6节）返回的颜色数组的整数索引。

从后端的角度来看，此函数可能被视为绘图 API 的一部分。然而，前端不需要实现它，因为它实际上是作为 draw_polygon() 的包装器在 misc.c 中集中实现的。

此函数可用于绘图和打印。

3.1.3 draw_rect_corners()

void draw_rect_corners(drawing *dr, int cx, int cy, int r, int col);

在正方形的角落绘制四个 L 形，以目标分划板的方式。这是后端用来显示键盘光标的便利函数（如果他们想要那种风格的光标）。

cx 和 cy 给出正方形中心的坐标。r 是正方形边长的一半，因此角落位于 (cx-r,cy-r)、(cx+r,cy-r)、(cx-r,cy+r) 和 (cx+r,cy+r)。

colour 是后端函数 colours()（第2.8.6节）返回的颜色数组的整数索引。

3.1.4 draw_line()

void draw_line(drawing *dr, int x1, int y1, int x2, int y2,
               int colour);

在拼图窗口中绘制一条直线。

x1 和 y1 给出线的一端的坐标。x2 和 y2 给出线的另一端的坐标。绘制的线包括这两个点。

colour 是后端函数 colours()（第2.8.6节）返回的颜色数组的整数索引。

一些平台可能对此函数执行抗锯齿。因此，不要假设你可以通过在背景颜色中绘制相同的线来擦除一条线；抗锯齿可能会导致消失线周围出现可感知的幽灵伪影。然而，水平和垂直线是像素完美的，不会被抗锯齿。

此函数可用于绘图和打印。

3.1.5 draw_polygon()

void draw_polygon(drawing *dr, const int *coords, int npoints,
                  int fillcolour, int outlinecolour);

在拼图窗口中绘制一个轮廓或填充的多边形。

coords 是一个 (2*npoints) 整数的数组，包含 npoints 个顶点的 x 和 y 坐标。x 和 y 坐标以交错顺序存储，每个顶点的 x 坐标在其 y 坐标之前。例如，第一个顶点的 x 和 y 坐标分别是 coords[0] 和 coords[1]。

fillcolour 和 outlinecolour 是后端函数 colours()（第2.8.6节）返回的颜色数组的整数索引。fillcolour 也可以是 -1，表示多边形应该只绘制轮廓。

由指定顶点列表定义的多边形首先用 fillcolour 填充（如果指定），然后用 outlinecolour 绘制轮廓。

outlinecolour 可能不是 -1；它必须是有效的颜色（前端允许通过断言强制执行这一点）。这是因为不同平台对填充多边形是否应包含其边界线存在分歧，因此绘制仅填充的多边形会产生不可移植的效果。如果你希望填充的多边形没有可见的轮廓，你必须将 outlinecolour 设置为与 fillcolour 相同。

一些平台可能对此函数执行抗锯齿。因此，不要假设你可以通过在背景颜色中绘制相同的多边形来擦除多边形。此外，准备好多边形由于这一点而超出其明显边界框一个像素；如果你真的需要它不这样做以避免干扰其他精细图形，你可能应该使用 clip()（第3.1.10节）。你可以依赖水平和垂直线不被抗锯齿。

此函数可用于绘图和打印。

3.1.6 draw_circle()

void draw_circle(drawing *dr, int cx, int cy, int radius,
                 int fillcolour, int outlinecolour);

在拼图窗口中绘制一个轮廓或填充的圆。

cx 和 cy 给出圆心的坐标。radius 给出其半径。圆的总水平像素范围从 cx-radius+1 到 cx+radius-1（含），垂直范围类似地围绕 cy。

fillcolour 和 outlinecolour 是后端函数 colours()（第2.8.6节）返回的颜色数组的整数索引。fillcolour 也可以是 -1，表示圆应该只绘制轮廓。

圆首先用 fillcolour 填充（如果指定），然后用 outlinecolour 绘制轮廓。

outlinecolour 可能不是 -1；它必须是有效的颜色（前端允许通过断言强制执行这一点）。这是因为不同平台对填充圆是否应包含其边界线存在分歧，因此绘制仅填充的圆会产生不可移植的效果。如果你希望填充的圆没有可见的轮廓，你必须将 outlinecolour 设置为与 fillcolour 相同。

一些平台可能对此函数执行抗锯齿。因此，不要假设你可以通过在背景颜色中绘制相同的圆来擦除圆。此外，准备好圆由于这一点而超出其明显边界框一个像素；如果你真的需要它不这样做以避免干扰其他精细图形，你可能应该使用 clip()（第3.1.10节）。

此函数可用于绘图和打印。

3.1.7 draw_thick_line()

void draw_thick_line(drawing *dr, float thickness,
                     float x1, float y1, float x2, float y2,
                     int colour)

在拼图窗口中绘制一条线，可以控制线的厚度。

x1 和 y1 给出线的一端的坐标。x2 和 y2 给出线的另一端的坐标。thickness 给出线的厚度，以像素为单位。

请注意，坐标和厚度是浮点数：这里使用连续坐标系。在这种情况下，能够以优于像素的精度寻址点很重要，因为否则无法正确表达具有奇数和偶数厚度的线的端点。

一些平台可能对此函数执行抗锯齿。粗线绘制操作影响的精确像素可能因平台而异，并且不提供特定的保证。实际上，即使水平或垂直线也可能被抗锯齿。

此函数可用于绘图和打印。

如果指定的厚度小于 1.0，则使用 1.0。这确保即使在小比例下细线也可见。

3.1.8 draw_text()

void draw_text(drawing *dr, int x, int y, int fonttype,
               int fontsize, int align, int colour,
               const char *text);

在拼图窗口中绘制文本。

x 和 y 给出一个点的坐标。此点与文本位置的关系由 align 指定，align 是水平和垂直对齐标志的按位或：

ALIGN_VNORMAL
表示 y 与文本的基线对齐。
ALIGN_VCENTRE
表示 y 与文本的垂直中心对齐。（实际上，它与正常大写文本的垂直中心对齐：在相同的 y 坐标处显示两个带有 ALIGN_VCENTRE 的文本片段将导致它们的基线彼此对齐，即使一个是上升符而另一个是下降符。）
ALIGN_HLEFT
表示 x 与文本的左端对齐。
ALIGN_HCENTRE
表示 x 与文本的水平中心对齐。
ALIGN_HRIGHT
表示 x 与文本的右端对齐。

fonttype 是 FONT_FIXED 或 FONT_VARIABLE，分别表示等宽或比例字体。（不能指定更多细节；这只会导致不同平台之间的可移植性问题。）

fontsize 是文本的所需大小，以像素为单位。此大小对应于文本的整体点大小，而不是任何内部尺寸（如大写高度）。

colour 是后端函数 colours()（第2.8.6节）返回的颜色数组的整数索引。

此函数可用于绘图和打印。

传递给此函数的文本的字符集由绘图对象指定，尽管它必须是 ASCII 的超集。如果拼图想要显示不包含在 ASCII 中的文本，它应该使用 text_fallback() 函数（第3.1.9节）来查询绘图对象以获取它想要的字符的适当表示。

3.1.9 text_fallback()

char *text_fallback(drawing *dr, const char *const *strings,
                    int nstrings);

此函数用于请求将 UTF-8 文本转换为绘图对象的 draw_text() 实现期望的字符编码。

输入是一个以 UTF-8 编码的字符串列表：nstrings 给出列表中字符串的数量，strings[0]、strings[1]、...、strings[nstrings-1] 是字符串本身。

返回的字符串（动态分配，使用完毕后必须释放）派生自列表中绘图对象期望能够可靠显示的第一个字符串；它将由该字符串转换为 draw_text() 期望的字符集组成。

绘图实现不需要处理 ASCII 以外的任何内容，但被允许假设某个字符串将被成功转换。因此，对此函数的每次调用都必须在列表中的某个位置（可能是最后一个元素）包含一个仅由 ASCII 组成的字符串，供任何无法处理其他内容的前端使用。

例如，如果拼图希望显示包含乘号的字符串（Unicode 中的 U+00D7，在 UTF-8 中由字节 C3 97 表示），它可能会这样做： 

static const char *const times_signs[] = { "\xC3\x97", "x" };
char *times_sign = text_fallback(dr, times_signs, 2);
sprintf(buffer, "%d%s%d", width, times_sign, height);
sfree(times_sign);
draw_text(dr, x, y, font, size, align, colour, buffer);
sfree(buffer);

这将在支持它的平台上绘制中间带有乘号的字符串，并在没有替代方案的地方回退到简单的 ASCII 'x'。

3.1.10 clip()

void clip(drawing *dr, int x, int y, int w, int h);

在拼图窗口中建立一个裁剪矩形。

x 和 y 给出裁剪矩形左上角像素的坐标。w 和 h 给出其宽度和高度。因此，矩形的水平范围从 x 到 x+w-1（含），垂直范围从 y 到 y+h-1（含）。（这些与 draw_rect() 的语义完全相同。）

此调用后，任何绘图操作都不会影响指定矩形外的任何内容。通过调用 unclip()（第3.1.11节）可以撤销此效果。裁剪矩形是像素完美的：矩形内的像素受绘图函数的影响与平常一样；矩形外的像素完全不受影响。

后端不应假设如果裁剪矩形被遗留下来，前端会自动清理它；这在当前前端上可能有效，但不应依赖。始终显式调用 unclip()。

此函数可用于绘图和打印。

3.1.11 unclip()

void unclip(drawing *dr);

撤销之前对 clip() 的调用的效果。此调用后，所有绘图操作将能够再次影响整个拼图窗口。

此函数可用于绘图和打印。

3.1.12 draw_update()

void draw_update(drawing *dr, int x, int y, int w, int h);

通知前端拼图窗口的矩形部分已被绘制并且需要更新。

x 和 y 给出更新矩形左上角像素的坐标。w 和 h 给出其宽度和高度。因此，矩形的水平范围从 x 到 x+w-1（含），垂直范围从 y 到 y+h-1（含）。（这些与 draw_rect() 的语义完全相同。）

后端重绘函数必须调用此函数来报告它对窗口所做的任何更改。否则，这些更改可能不会立即可见，然后可能会在不可预测的后续时间出现，例如下次窗口被覆盖并重新暴露时。

此函数仅在绘图时重要。它也可以在打印时调用，但这样做不是强制性的，并且没有效果。（因此，如果你在绘图和打印例程之间有共享的代码，该代码可以安全地调用 draw_update()。）

3.1.13 status_bar()

void status_bar(drawing *dr, const char *text);

将游戏状态栏中的文本设置为 text。文本从提供的缓冲区复制，因此调用者在使用后可以自由释放或修改缓冲区。

（此函数不是一个绘图函数，但它与绘图 API 共享一个属性，即它只能从后端重绘函数内部调用。并且它由前端通过 drawing_api 函数指针表实现。所以这是记录它的最佳位置。）

提供的文本在传递给前端之前会通过中端进行可选重写；如果游戏是计时的，中端会在前面加上当前游戏时间（将来如果看起来是个好主意，可能会执行其他重写）。

此函数仅用于绘图；绝不能在打印期间调用它。

3.1.14 Blitter 函数

本节描述一组相关函数，用于保存和恢复拼图窗口的一部分。这最常用于实现涉及在窗口周围拖动拼图元素的用户界面：在每次调用 redraw() 结束时，如果当前正在拖动对象，后端会保存该位置下的窗口内容，然后绘制被拖动的对象，在下一次 redraw() 开始时，它做的第一件事就是恢复背景。

前端定义了一种称为 blitter 的不透明类型，能够存储指定大小的矩形区域。

Blitter 函数仅用于绘图；绝不能在打印期间调用它们。

3.1.14.1 blitter_new()

blitter *blitter_new(drawing *dr, int w, int h);

创建一个新的 blitter 对象，存储大小为 w 乘 h 像素的矩形。返回指向 blitter 对象的指针。

Blitter 对象最好存储在 game_drawstate 中。创建它们的好时机是在 set_size() 函数中（第2.8.5节），因为此时你首先知道它们需要保存的矩形有多大。

3.1.14.2 blitter_free()

void blitter_free(drawing *dr, blitter *bl);

处理 blitter 对象。最好在 free_drawstate() 中调用。（但是，在尝试释放 blitter 对象之前，请检查它是否为 NULL；draw state 可能在创建和释放之间从未调用过 set_size()。）

3.1.14.3 blitter_save()

void blitter_save(drawing *dr, blitter *bl, int x, int y);

这是一个真正的绘图 API 函数，因为它只能在游戏重绘例程内部调用。它将拼图窗口的矩形部分保存到指定的 blitter 对象中。

x 和 y 给出保存的矩形左上角的坐标。矩形的宽度和高度是创建 blitter 对象时指定的。

此函数需要处理 x 和 y 超出范围的情况并做正确的事情。（正确的事情可能意味着保存 blitter 矩形与拼图窗口可见区域重叠的任何部分。）

3.1.14.4 blitter_load()

void blitter_load(drawing *dr, blitter *bl, int x, int y);

这是一个真正的绘图 API 函数，因为它只能在游戏重绘例程内部调用。它从指定的 blitter 对象中恢复拼图窗口的矩形部分。

x 和 y 给出要恢复的矩形左上角的坐标。矩形的宽度和高度是创建 blitter 对象时指定的。

此函数需要处理 x 和 y 超出范围的情况并做正确的事情。（正确的事情可能意味着恢复 blitter 矩形与拼图窗口可见区域重叠的任何部分。）

如果此函数在之前从部分超出范围的矩形保存的 blitter 上调用，那么在保存时不可见的保存位图部分是未定义的。如果 blitter 被恢复到不同的位置，使这些部分可见，则对绘图区域的影响是未定义的。

3.1.15 print_mono_colour()

int print_mono_colour(drawing *dr, int grey);

此函数在打印期间为简单的单色分配颜色索引。

grey 必须是 0 或 1。如果 grey 是 0，返回的颜色是黑色；如果 grey 是 1，颜色是白色。

3.1.16 print_grey_colour()

int print_grey_colour(drawing *dr, float grey);

此函数在打印期间为灰度颜色分配颜色索引。

grey 可以是 0（黑色）到 1（白色）之间的任何数字；例如，0.5 表示中等灰色。

所选颜色将根据打印机的半色调能力进行渲染。

3.1.17 print_hatched_colour()

int print_hatched_colour(drawing *dr, int hatch);

此函数分配一个不表示字面颜色的颜色索引。相反，以此颜色着色的区域将用平行线进行阴影处理。hatch 参数定义了应该用于此颜色的阴影类型：

HATCH_SLASH
此颜色将由向右倾斜 45 度的线进行阴影处理。
HATCH_BACKSLASH
此颜色将由向左倾斜 45 度的线进行阴影处理。
HATCH_HORIZ
此颜色将由水平线进行阴影处理。
HATCH_VERT
此颜色将由垂直线进行阴影处理。
HATCH_PLUS
此颜色将由交叉的水平线和垂直线进行阴影处理。
HATCH_X
此颜色将由交叉的对角线进行阴影处理。

定义为使用阴影的颜色不得用于绘制线条或文本；它们只能用于填充区域。也就是说，它们可以用作 draw_circle() 和 draw_polygon() 的 fillcolour 参数，以及 draw_rect() 的颜色参数，但不得用作 draw_circle() 或 draw_polygon() 的 outlinecolour 参数，或与 draw_line() 或 draw_text() 一起使用。

3.1.18 print_rgb_mono_colour()

int print_rgb_mono_colour(drawing *dr, float r, float g,
                          float b, float grey);

此函数在打印期间为完全指定的 RGB 颜色分配颜色索引。

r、g 和 b 各自可以在 0 到 1 的范围内。

如果仅以黑白打印，这些值将被忽略，而是根据 'grey' 参数使用纯黑色或纯白色。（回退颜色与 print_mono_colour(grey) 分配的颜色相同。）

3.1.19 print_rgb_grey_colour()

int print_rgb_grey_colour(drawing *dr, float r, float g,
                          float b, float grey);

此函数在打印期间为完全指定的 RGB 颜色分配颜色索引。

r、g 和 b 各自可以在 0 到 1 的范围内。

如果仅以黑白打印，这些值将被忽略，而是使用由 grey 参数给出的灰度。（回退颜色与 print_grey_colour(grey) 分配的颜色相同。）

3.1.20 print_rgb_hatched_colour()

int print_rgb_hatched_colour(drawing *dr, float r, float g,
                             float b, float hatched);

此函数在打印期间为完全指定的 RGB 颜色分配颜色索引。

r、g 和 b 各自可以在 0 到 1 的范围内。

如果仅以黑白打印，这些值将被忽略，而是使用由 hatch 参数给出的交叉阴影形式；有关此参数的可能值，请参见 第3.1.17节。（回退颜色与 print_hatched_colour(hatch) 分配的颜色相同。）

3.1.21 print_line_width()

void print_line_width(drawing *dr, int width);

此函数用于设置打印期间绘制的线条的粗细。它在绘图中没有意义：draw_line()、draw_circle 和 draw_polygon() 绘制的所有线条都是一个像素的粗细。然而，在打印中，没有明确的像素定义，因此必须显式指定线条宽度。

线条宽度在通常的坐标系中指定。然而，请注意，这只是一个提示：中央打印系统可能会根据用户请求或打印机功能选择改变线条粗细。

3.1.22 print_line_dotted()

void print_line_dotted(drawing *dr, bool dotted);

此函数用于在打印期间切换虚线的绘制。它在绘图期间不受支持。

将 'dotted' 设置为 true 意味着未来由 draw_line()、draw_circle 和 draw_polygon() 绘制的线条将是虚线。将其设置为 false 意味着它们将是实线。

一些前端可能对虚线的宽度施加限制。如果前端需要，通过此前端请求虚线将覆盖任何线条宽度请求。

3.2 前端实现的绘图 API

本节描述前端实现的函数指针形式的绘图 API。

（不仅是平台特定的前端实现此 API；平台无关的模块 ps.c 也提供了一个实现，用于输出 PostScript。因此，任何想要进行 PS 打印的平台都可以以最小的麻烦做到这一点。）

以下条目（除了 version 字段外）都描述了名为 drawing_api 的结构中的函数指针字段。每个函数都接受一个“drawing *”指针，该指针又包含一个“void *handle”成员，前端可以访问该成员并在内部将其转换回更有用的类型。由于这是前端经常要做的事情，因此提供了 GET_HANDLE_AS_TYPE() 宏来为你执行此操作（参见 第 5.9.20 节）。

适合传递给后端重绘或打印函数的绘图 对象（drawing *) 是通过将 drawing_api 和一个“void *”传递给函数 drawing_new() 来构造的（参见 第 3.3.1 节）。

3.2.1 version

int version;

这是一个整数，给出前端实现的绘图 API 的版本。对于本文档中描述的 API 版本，此字段应为 1。

有时，绘图 API 可能会发生破坏性更改，例如引入新函数或更改现有函数的语义。此版本字段的目的是在发生此类破坏性更改时向前端作者发出信号。由于此类更改可能会对下游前端造成干扰，因此应谨慎进行。

3.2.5 draw_polygon()

此函数在打印期间，在每页开始时调用。它提供页码（从 1 而不是 0 开始编号，因此适合在用户可见的上下文中使用）。

3.2.20 begin_puzzle()

void (*begin_puzzle)(drawing *dr, float xm, float xc,
                     float ym, float yc, int pw, int ph, float wmm);

此函数在打印期间，在页面上打印单个拼图之前调用。它指定拼图在页面上的大小和位置。

xm 和 xc 指定拼图在页面上的水平位置，作为页面宽度的线性函数。前端应将页面宽度乘以 xm，加上 xc（以毫米为单位），并使用得到的 x 坐标作为拼图的左边缘。

类似地，ym 和 yc 指定拼图的垂直位置作为页面高度的函数：页面高度乘以 ym，加上 yc 毫米，等于从页面顶部到拼图顶部的所需距离。

（需要这种笨拙的机制是因为并非所有打印系统都能将页面大小传达回软件。例如，PostScript 后端写出的 PS 在打印时通过调用“clippath”来确定页面大小，并在其中居中拼图。因此，完全相同的 PS 文件可以在 A4 或 US Letter 纸上工作，而无需本地配置，这简化了事情。）

pw 和 ph 给出绘图 API 坐标中拼图的大小。打印系统随后将调用拼图自己的打印函数，该函数又将调用绘图 API 函数，期望有一个 pw 乘 ph 单位的区域可用于绘制拼图。

最后，wmm 给出拼图的所需宽度（以毫米为单位）。（预期会保持纵横比，因此如果还需要所需的拼图高度，则可以计算为 wmm*ph/pw。）

不提供打印服务的此 API 实现可以将此函数指针定义为 NULL；除非尝试打印，否则永远不会调用它。

3.3 前端调用的绘图 API

drawing.c 中提供了少量函数，前端需要 调用 这些函数，而不是帮助实现它们。本节描述这些函数。

3.3.1 drawing_new()

drawing *drawing_new(const drawing_api *api, midend *me,
                     void *handle);

此函数创建一个绘图对象。它传递一个 drawing_api，这是一个包含函数指针和版本字段的结构；还传递一个“void *”句柄。句柄存储在绘图对象中，当调用每个函数指针时，可以从那里访问它。

midend 参数用于重写状态栏内容：status_bar()（参见 第 3.1.13 节）必须调用中端中的一个函数，该函数可能会重写状态栏文本。如果绘图对象仅用于打印，或者已知游戏不会调用 status_bar()，则此参数可以为 NULL。

如果 drawing_api 的 version 字段与预期版本不匹配（参见 第 3.2.1 节 了解预期版本号），则会产生致命错误。

3.3.2 drawing_free()

void drawing_free(drawing *dr);

此函数释放绘图对象。请注意，“void *”句柄不会被释放；如果需要清理，必须由前端完成。

3.3.3 print_get_colour()

void print_get_colour(drawing *dr, int colour,
                      bool printing_in_colour,
                      int *hatch, float *r, float *g, float *b);

此函数由绘图 API 函数的实现在打印上下文中被调用时调用。它接受颜色索引作为输入，并返回后端请求的颜色描述。

printing_in_colour 为 true 当且仅当实现在彩色打印。如果所讨论的颜色是用黑白回退值指定的，这将改变返回的结果。

如果颜色应通过阴影线渲染，*hatch 将填充所需的阴影线类型。有关此整数可以采用的值的详细信息，请参见 第 3.1.16 节。

如果颜色应渲染为纯色，*hatch 将被赋予负值，并且 *r、*g 和 *b 将填充所需颜色的 RGB 值（如果以彩色打印），或全部填充灰度值（如果以黑白打印）。

3.3.4 draw_polygon_fallback()

void draw_polygon_fallback(drawing *dr,
                           const int *coords, int npoints,
                           int fillcolour, int outlinecolour);

此函数旨在供没有原生多边形填充原语的前端使用。其签名和语义与 draw_polygon()（第 3.1.5 节）完全相同；然而，它不是由前端实现的，而是作为使用扫描线算法的中间件实现，该算法产生一系列 draw_line() 调用，具有填充和勾勒所需多边形的效果。

尽管前端可以选择直接调用此函数（例如，从 draw_polygon() 的存根实现中），但此函数旨在由前端 drawing_api 的 draw_polygon 字段 指向；也就是说，没有多边形填充原语的前端应将 drawing_api 的 draw_polygon 字段设置为 draw_polygon_fallback。

采用这种相当笨拙的方法来使用此回退函数（而不是在 drawing_api 中将 draw_polygon 简单地设置为 NULL）的动机是，它允许链接时优化编译器在提供自己的 draw_polygon() 的平台上修剪此函数的实现，因为此函数在这些平台上永远不会被引用。（但此函数仍然在所有平台上无条件编译，从而保护它免受位腐烂。）
