第 2 章：通用功能

本章描述所有游戏共有的功能。

2.1 通用操作

除了任何特定于游戏的操作外，这些操作都可以从 “游戏”菜单和通过 键盘快捷键获得。

（在 Mac OS X 上，为了符合本地用户界面标准，这些操作位于 “文件”和 “编辑”菜单中。）

新游戏（'N'，Ctrl+'N'）
开始一个新游戏，初始状态随机。
重新开始游戏
将当前游戏重置为其初始状态。（这可以撤销。）
加载
从磁盘上的文件加载保存的游戏。
保存
将游戏的当前状态保存到磁盘上的文件。

加载和保存操作保留您的整个游戏历史（因此您可以保存、重新加载，并且仍然可以撤销和重做保存之前所做的事情）。

打印
在支持的地方（目前仅在 Windows 上），会弹出一个对话框，允许您打印任意数量的从当前参数随机生成的拼图，可选地包括当前拼图。（当然，仅适用于有意义打印的拼图 —— 很难想象 Fifteen 的合理可打印表示！）
撤销（'U'，Ctrl+'Z'，Ctrl+'_'，'*'）
撤销单个移动。（您可以撤销移动回到会话开始。）
重做（'R'，Ctrl+'R'，'#'）
重做之前撤销的移动。
复制
将游戏的当前状态以文本格式复制到剪贴板，以便您可以将其粘贴到（例如）电子邮件客户端或网络留言板中，如果你正在与其他人讨论游戏。（并非所有游戏都支持此功能。）
解决
将拼图立即转换为其已解决状态。对于某些游戏（Cube），此功能完全不支持，因为它没有特别的用处。对于其他游戏（如 Pattern），已解决状态可以为您提供信息，如果你根本看不到解决方案如何存在，或者你想知道你在哪里犯了错误。对于其他游戏（如 Sixteen），自动解决方案不会告诉你如何到达解决方案，但它确实提供了一种快速到达那里的有用方法，以便你可以实验固定动作和变换。

一些游戏（如 Solo）能够解决你从其他地方输入的游戏 ID。其他游戏（如 Rectangles）无法解决它们自己没有发明的游戏 ID，但当它们确实发明了游戏 ID 时，它们已经知道解决方案是什么。还有其他游戏（Pattern）可以解决一些外部游戏 ID，但只有在它们不太困难的情况下。

“解决”命令将已解决状态添加到拼图的撤销链的末尾。换句话说，如果你在看到答案后想回到自己解决它，你只需按撤销。

退出（'Q'，Ctrl+'Q'）
完全关闭应用程序。
首选项
在支持的地方，会弹出一个对话框，允许你配置关于特定游戏的个人首选项。其中一些首选项将特定于特定游戏；其他则将对所有游戏通用。

所有游戏共有的一个选项允许你关闭一键快捷键，如 'N' 用于新游戏或 'Q' 用于退出，以减少意外击中它们的机会。你仍然可以使用 Ctrl 键访问相同的快捷键。

2.2 使用 游戏 ID 指定游戏

有两种方法可以从拼图中保存游戏规范并在以后重新创建它，或在其他人的同一拼图副本中重新创建它。

“特定”和 “随机种子”选项从 “游戏”菜单（或在 Mac OS X 上的“文件”菜单）中各自显示一段文本（“游戏 ID”），足以在以后重建完全相同的游戏。

你可以稍后将这些文本中的任何一个输入回程序（通过相同的“特定”或“随机种子”菜单选项），它将重新创建相同的游戏。你也可以将其中任何一个用作 命令行参数（在 Windows 或 Unix 上）；有关更多详细信息，请参见 第 2.4 节。

这两种形式之间的区别在于，描述性游戏 ID 是游戏 初始状态的字面描述，而随机种子只是作为用于创建拼图的随机数生成器的输入的一段任意文本。这意味着：

  • 在许多拼图中，描述性游戏 ID 往往更长（尽管有些，如 Cube（第 4 章），只需要很短的描述）。因此，随机种子通常是一种更快的方式来记下你当前正在玩的拼图，或者告诉其他人，以便他们可以玩与你相同的拼图。
  • 任何文本都是有效的随机种子。自动生成的种子是十五位数字，但任何东西都可以；你可以输入你的全名，或你刚刚编造的一个词，并且会从它生成一个有效的拼图。这为两个或更多人竞相完成同一拼图提供了一种方法：你想出一个随机种子，然后每个人同时输入它，没有人因为在其他人之前看到生成的拼图而有优势。
  • 通常可以将来自其他来源的拼图（如报纸上的“非ograms”或“数独”）转换为适合与这些程序一起使用的描述性游戏 ID。
  • 如果你在不同版本的拼图程序中使用随机种子，不能保证它们会产生相同的结果。这是因为生成算法可能在代码的后续版本中得到改进或修改，因此在给定相同的随机数序列时会产生不同的结果。如果你不确定它是否会在与你的程序相同版本上使用，请使用描述性游戏 ID。

    （使用“关于”菜单选项找出程序的版本号。在不同平台上运行的具有相同版本号的程序仍然应该是随机种子兼容的。）

描述性游戏 ID 以一段文本开始，该文本编码当前游戏的 参数（如网格大小）。然后是一个冒号，之后是游戏初始状态的描述。随机种子以类似的参数字符串开始，但随后包含一个哈希符号后跟任意数据。

如果你输入描述性游戏 ID，程序将无法向你显示生成它的随机种子，因为它不是从随机种子生成的。然而，如果你输入随机种子，程序将能够向你显示从该随机种子派生的描述性游戏 ID。

请注意，两种形式之间的游戏参数字符串并不总是相同的。对于某些游戏，随机种子会提供参数数据，而描述性游戏 ID 中不包含这些数据。这是因为该参数信息仅与生成拼图网格相关，在玩它们时并不重要。因此，例如，Solo（第 11 章）中的难度级别在描述性游戏 ID 中没有提及。

如果你输入游戏 ID，这些附加参数也不会永久设置。例如，假设你将 Solo 设置为“高级”难度级别，然后朋友需要你帮助解决“简单”拼图；朋友读出指定“简单”难度的随机种子，你输入它。程序会为你生成朋友遇到困难的同一个“简单”网格，但一旦你完成玩它，当你要求新游戏时，它会自动回到之前设置的“高级”难度。

2.3 “类型”菜单

如果存在，“类型”菜单可能包含 预设游戏设置的列表。选择其中一个将使用指定的参数开始一个新的随机游戏。

“类型”菜单还可能包含一个 “自定义”选项，允许你微调游戏 参数。可用的参数特定于每个游戏，并在以下部分中描述。

2.4 在 命令行上指定游戏参数

（本节不适用于 Mac OS X 版本。）

此集合中的游戏故意从不将信息保存到它们运行的计算机上：它们没有高分表，也没有保存的首选项。（这是因为我预计至少有一些人会在工作中玩它们，那些人可能会喜欢留下尽可能少的证据！）

然而，如果你确实希望安排这些游戏之一默认到特定的参数集，你可以在命令行上指定它们。

最简单的方法是使用“类型”菜单（见 第 2.3 节）设置你想要的参数，然后从“游戏”或“文件”菜单中选择“随机种子”（见 第 2.2 节）。“游戏 ID”框中的文本将由两部分组成，用哈希分隔。第一部分代表游戏参数（例如，游戏区域的大小，以及你使用“类型”菜单设置的任何其他内容）。

如果你仅使用该参数字符串在命令行上运行游戏，它将以你指定的设置启动。

例如：如果你运行 Cube（见 第 4 章），从“类型”菜单中选择“八面体”，然后进入游戏 ID 选择，你将看到一个形式为“o2x2#338686542711620”的字符串。只取哈希之前的部分（“o2x2”），并在命令行上用该文本启动 Cube：“PREFIX-cube o2x2”。

如果你将整个游戏 ID 复制到命令行，游戏将以所描述的特定游戏启动。这偶尔是启动特定游戏 ID 的更方便的方式，而不是将其粘贴到游戏 ID 选择框中。

（你也可以使用“特定”菜单选项而不是“随机种子”来检索编码的游戏参数，但如果你这样做，一些选项，如 Solo 中的难度级别，将缺失。有关更多详细信息，请参见 第 2.2 节。）

2.5 Unix 命令行选项

（本节仅适用于 Unix 端口。）

除了能够在命令行上指定游戏参数（见 第 2.4 节）外，还有各种其他选项：

--game
--load
这些选项分别确定命令行参数是被视为指定游戏参数还是要 加载的 保存文件。只应指定一个。如果未指定这些选项中的任何一个，则根据参数的格式进行猜测。
--generate n
如果指定了此选项，将不会显示拼图，而是会在标准输出上 生成并打印一些描述性游戏 ID。这对于在不使用前端的情况下访问游戏生成算法很有用。

如果在命令行上指定了游戏参数，它们将用于生成游戏 ID；否则将使用默认的参数集。

此选项最常见的用途是与 --print 结合使用，在这种情况下其行为略有不同；见下文。

--delete-prefs
此选项导致拼图删除存储其用户首选项的配置文件（如果有）。
--print wxh
如果指定了此选项，将不会显示拼图，而是将一个或多个未解决拼图的打印表示以 PostScript 格式发送到标准输出。

在每页拼图上，将有 w 个横向和 h 个纵向。如果拼图数量超过 w×h，将打印多页。

如果还指定了 --generate，则发明的游戏 ID 将用于生成打印输出。否则，标准输入上需要游戏 ID 列表（可以是描述性或随机种子；见 第 2.2 节），格式与 --generate 产生的格式相同。

例如： 

PREFIX-net --generate 12 --print 2x3 7x7w | lpr

将生成两页打印的 Net 拼图（每页将有一个 7×7 的包装网格），并将输出通过管道传递给 lpr 命令，在许多系统上，这将把它们发送到实际的打印机。

有各种其他影响打印的选项；见下文。

--save file-prefix [ --save-suffix file-suffix ]
如果指定了此选项，将不会显示拼图，而是将一个或多个未解决拼图的保存游戏文件写入从提供的前缀和/或后缀构造的文件中。

如果还指定了 --generate，则发明的游戏 ID 将用于生成打印输出。否则，标准输入上需要游戏 ID 列表（可以是描述性或随机种子；见 第 2.2 节），格式与 --generate 产生的格式相同。

例如： 

PREFIX-net --generate 12 --save game --save-suffix .sav

将生成十二个 Net 保存游戏文件，名称从 game0.sav 到 game11.sav。

--version
打印有关游戏的版本信息，然后退出。

以下选项仅在同时指定 --print 时才有意义：

--with-solutions
填充未解决拼图的页面集之后将是这些拼图的解决方案。
--scale n
调整打印时每个拼图的大小。较大的数字使拼图更大；默认值为 1.0。
--colour
拼图将以彩色打印，而不是黑白（如果拼图支持）。
